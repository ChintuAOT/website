Half Scale Handle



Hand-tuned linear feedback controller on initial feasibility model

Over a holiday weekend in 2023, I caught the multi-link robot bug again and had the idea to build a half-scale version of Boston Dynamics' Handle robot. I had been looking for something more interesting to do with my custom actuators than just an arm or a robot dog (seems like everyone is building their own these days). Handle has long been my favorite robot for a couple reasons and the wheeled-legged nature of it's design means controls are fairly reasonable. The superficial goal for this project is to do a shot-by-shot remake of the original BD video.

// Actuators


Two assembled V1 actuators

The inception point for this project was wanting to design my own QDD actuator, then shortly after, wanting a more complete project to motivate me and force some requirements. If I was going for pure performance and dev speed I would shell out a few more hundred dollars for a COTS mini-cheetah actuator clone, or do a driver swap on something cheaper, but that wouldn't be as fun and I would learn far less.

Exploded cad for V2 of the actuator


The main unique design point behind these actuators is using a 3D printed ring gear. This cuts out a large amount of cost while allowing for a more integrated design. I ran a few FEA sims assuming solid nylon and was fairly impressed at the performance. There are multiple teeth meshed per planet on the ring gear so the resulting force per tooth ends up being pretty reasonable. There are obviously wear concerns, but I have not seen any noticeable degradation and the nature of hobby projects means I might not run them enough to reach that point.

// Arm development


Rough arm CAD

My first hardware goal is building the arm as it is a mostly self-contained project. I have completed the first two links + actuators and have run rough position control. I have yet to implement a proper velocity control loop on the drivers so there is no controlled damping and therefore movement is still a bit jumpy. I am close to adding the wrist and suction gripper and beginning to do some manipulation testing. The actuators seem to work fairly well and I haven't seen any degradation using the printed ring gear yet.

Slow sine position goals on arm.

// Perception


Alongside the hardware and controls for this project, I want to write some basic manipulation software to allow it to semi-autonomously pick and stack boxes. I am starting out by building cardboard box segmentation network with the goal of generating suction gripper targets. The network is currently a Mask-RCNN network fine-tuned using synthetic data generated in Blender. I am using an OAK-D Lite for depth sensing, but am tempted to get one of the OAK cameras with an active IR projector.
